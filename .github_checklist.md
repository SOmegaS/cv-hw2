# GitHub Checklist для сдачи ДЗ

## Перед загрузкой на GitHub

### 1. Создать репозиторий
```bash
git init
git add .
git commit -m "Initial commit: DETR HW2 & HW2.5"
git remote add origin https://github.com/YOUR_USERNAME/detr-hw2.git
git push -u origin main
```

### 2. Что загружать

✅ **ОБЯЗАТЕЛЬНО:**
- `src/` - весь код
- `requirements.txt`
- `README.md`
- `RESULTS.md`
- `RESULTS_HW25.md`
- `HW2.md` (задание)
- `examples/`
- `.gitignore`
- Скрипты: `setup.sh`, `download_coco.sh`, `full_pipeline.sh`, etc.

✅ **ЖЕЛАТЕЛЬНО:**
- `visualizations/training_curves.png`
- `visualizations/synthetic_samples/` (примеры, ~10 MB)
- Несколько примеров из `visualizations/predictions/` (5-10 штук)

❌ **НЕ ЗАГРУЖАТЬ:**
- `data/` - слишком большой (~20 GB)
- `outputs/` - слишком большой (~5 GB)
- `venv/` - виртуальное окружение
- `*.log` - логи
- `.pyc`, `__pycache__/` - Python кеш

### 3. README должен содержать

- Описание задания
- Инструкции по установке
- Примеры команд
- Результаты (или ссылка на RESULTS.md)
- Структура проекта

### 4. Дополнительно (для оценки)

Можно создать отдельный файл `RESULTS_SUMMARY.md`:
- Таблицы метрик
- Графики (screenshots или ссылки)
- Примеры визуализаций
- Выводы

### 5. Для демонстрации

Если нужно показать TensorBoard логи:
- Загрузить на TensorBoard.dev:
  ```bash
  tensorboard dev upload --logdir outputs/full_run/logs
  ```
- Добавить ссылку в README

## Проверка перед сдачей

- [ ] Весь код на месте
- [ ] README с инструкциями
- [ ] RESULTS.md с таблицами
- [ ] RESULTS_HW25.md готов
- [ ] requirements.txt корректный
- [ ] Примеры визуализаций
- [ ] .gitignore настроен
- [ ] Репозиторий публичный (или преподаватель добавлен)
